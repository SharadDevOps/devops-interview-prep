# FinTrust Bank — Corporate Case Study

## Lead DevOps Engineer Interview Simulation

This case study is based on a real-world banking cloud transformation scenario.

### Company: FinTrust Bank
- Industry: Banking & FinTech (India)
- Scale: 18 million customers, 4 million daily transactions
- Challenge: Migrate 240 on-prem workloads to Azure, build CI/CD, achieve 99.99% availability

### Technologies Covered
- Azure Migrate (6 R's strategy)
- Terraform (modules from scratch)
- GitHub Actions (OIDC, reusable workflows)
- AKS (cluster management, upgrades, autoscaling)
- Prometheus + Grafana + ELK (observability)
- Azure Key Vault (secrets management)
- ExpressRoute (hybrid connectivity)
- Hub-Spoke VNet topology

### Submission Questions
1. Migration Strategy — Finacle CBS vs Internet Banking Portal
2. Terraform Architecture — multi-env tfstate management
3. GitHub Actions Pipeline — payment gateway with dual approval
4. Incident Response — payment API error spike at 11:30pm
5. Design Conflict — Network Policies vs dev team needs
